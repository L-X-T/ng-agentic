Getting Started with the Workshop

The on-ramp before the hands-on labs. In a few minutes you'll get the workspace's dependencies installed and running, and – just as important – pick the project you'll work on throughout the rest of the workshop. About this lab

This is Lab 00 – the warm-up. Before you reproduce the agentic tooling in Lab 01, two things need to be in place: the project must install and run on your machine, and you need a project of your own to apply everything to. A workshop sticks far better when every exercise nudges a real goal forward instead of a throwaway demo.

Why this matters

Agentic engineering is learned by doing. If you carry a small, personal project through the labs, each new piece of tooling – the linter, the style guide, the MCP servers, the agent itself – lands on code you understand and want to improve. That context is what turns "I followed the steps" into "I can do this on my own work."

Learning objectives

  • Install the workspace's dependencies so the app builds and serves.
  • Confirm your toolchain is ready: Node, a package manager, and at least one AI coding agent.
  • Choose a small project or work package to carry through the workshop.
  • Write down a concrete goal to aim for – realistic or not.

Prerequisites

  • Node.js 22+ installed (node -v to check).
  • A package manager – npm ships with Node; this workspace standardizes on pnpm.
  • Git installed and configured (name + email).
  • At least one AI coding agent installed (e.g. Claude Code or Codex).
  • A terminal and your editor of choice (VS Code / JetBrains recommended).
How to work through this lab

Each step has a Goal, some background, a Your task box, a collapsible Solution where useful, and a Checkpoint to confirm you're done. Steps 3 and 4 have no single "right answer" – the deliverable is your project and your goal, written down.

Steps

  2. Verify your setup runs
  3. Choose your workshop project
  4. Write down your goal

1 Install the project dependencies

Goal: pull down everything the workspace needs so it can build, serve, lint and format.

You've cloned (or downloaded) the ng-agentic repository. Before anything will run, the dependencies listed in package.json have to be installed into node_modules.

Your task

From the repository root, install the dependencies.

Show solution 
shellnpm install

That's it – npm reads package.json and downloads everything into node_modules/.

Tip – this workspace uses pnpm

The project standardizes on pnpm and commits a pnpm-lock.yaml. If you have pnpm installed, prefer pnpm install so you resolve the exact same dependency versions everyone else does (and you avoid creating a stray package-lock.json). Don't have pnpm yet? npm i -g pnpm or enable Corepack with corepack enable.

Why first

Every later command – start, build, lint, format – depends on these packages being present. Get this green before moving on.

Checkpoint: the install finishes without errors and a node_modules/ folder exists at the repo root.

2 Verify your setup runs

Goal: prove the app serves and your toolchain is ready before you build on top of it.

Your task

  1. Start the dev server and open the app in a browser.
  2. Check your Node and package-manager versions.
  3. Confirm at least one AI coding agent is installed and can open this folder.
Show solution 
shellnpm start          # serves on http://localhost:4200
node -v            # expect v22 or newer
npm -v             # or: pnpm -v

Open http://localhost:4200 – you should see the running app. Stop the server with Ctrl + C when you're done.

Then make sure your agent is available, e.g. claude or codex on your PATH, and that it can read this repository.

If it doesn't start

A failed npm start almost always traces back to Step 1 (re-run the install) or a Node version that's too old (node -v must be 22+). Fix it here – every following lab assumes a working app.

Checkpoint: the app loads at localhost:4200, node -v reports 22+, and your agent can open the project folder.

3 Choose your workshop project

Goal: decide what you'll actually work on for the rest of the workshop – before we dive into the tooling.

The labs are far more valuable when you apply them to something real instead of a disposable example. So pick a project now. It doesn't have to be big – a focused work package is perfect.

Pick something that fits these

Small. Define a small project or a single work package – something you could meaningfully move forward in a workshop, not a quarter.
Yours. Relate it to your work, a hobby, or a side project – the more you care, the more the labs stick.

Your task

Write down a short description of the project you'll carry through the workshop. Capture:

  • Name – a working title for the project or work package.
  • What it is – one or two sentences on what you're building or improving.
  • Why it's yours – is it from work, a hobby, or a side project? Why does it matter to you?
  • Scope – what's deliberately in, and what you'll leave out for now.
Show solution – example project briefs

Any of these would make a great workshop project:

  • Recipe box – a side project: a small Angular app to save and search my favourite recipes. Scope: list + detail + add form. Out of scope: accounts, sync.
  • Team on-call dashboard – from work: a single page that shows who's on call this week. Scope: read-only view from a JSON feed. Out of scope: editing the schedule.
  • Board-game score tracker – a hobby: log scores for game nights and show a leaderboard. Scope: add a game, add scores, show standings. Out of scope: stats over time.

Notice the pattern: one clear feature set, a named reason it's yours, and an explicit "out of scope" so it stays small.

Checkpoint: you have a written, one-paragraph brief for a small project that's connected to your work, a hobby, or a side project.

4 Write down your goal

Goal: turn the project into a concrete target you can aim at – realistic or not.

A project is a direction; a goal is a finish line. Writing it down – even an ambitious or unrealistic one – gives every lab something to push toward and makes it obvious when you've made progress.

Your task

Write a single sentence that states what you want to have achieved by the end of the workshop. Then add an optional "stretch" version that's deliberately bigger than what feels realistic.

Show solution – example goals
ProjectRealistic goalStretch goal (realistic or not)
Recipe box Have an agent build the list + detail views following the project conventions. Ship a fully accessible, tested app with search and tagging.
On-call dashboard Render the current on-call person from a static JSON feed. Live data, dark mode, and a one-click "page me" button.
Score tracker Add scores and show a leaderboard for one game. Multi-game stats, charts, and shareable links.

Keep the goal visible while you work the labs – it's your reference for whether the agentic setup is actually moving your project forward.

Tip

Don't over-think it. A goal you can revise later beats a perfect goal you never write down. The point is to have a target before Lab 01.

Checkpoint: you've written one sentence stating your goal (and, optionally, a stretch goal) for the workshop.
