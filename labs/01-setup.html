Hands-on Lab · Agentic Engineering

Set up an Angular project for Agentic Engineering

In this lab you will reproduce, step by step, the tooling and conventions used in the ng-agentic workspace inside the project you chose in Lab 00. If you did not bring a suitable project, scaffold a fresh Angular 22 workspace and use that as your workshop project. Either way, the target is a codebase that is safe and productive to work on together with AI coding agents – Prettier, Husky, ESLint, lint-staged, a shared style guide, per-agent instruction files, and MCP servers.

Angular v22 Package manager pnpm 10 Node 22+ (target 26) Est. time 75–120 min Level intermediate

About this lab

"Agentic engineering" means treating AI coding agents (Claude Code, OpenAI Codex, Gemini, JetBrains Junie, GitHub Copilot, Cursor, Windsurf, …) as first-class teammates. For that to work, the project you chose in Lab 00 has to encode its rules: how code should look, what is allowed, what the agent may read, and which tools it can reach. By the end you'll have a workspace where any agent – and any human – follows the same standards automatically.

Why this matters

An agent with no guardrails produces inconsistent "AI slop". An agent given a precise style guide, a linter that auto-fixes, a pre-commit hook that enforces formatting, and an .aiignore that hides secrets becomes a reliable contributor. You are building that environment.

Learning objectives

  • Apply the setup to the project you chose in Lab 00, or scaffold a fallback Angular workspace if needed.
  • Enforce formatting and linting automatically with Prettier, ESLint, Husky and lint-staged.
  • Author a shared, multi-file style guide that both humans and agents follow.
  • Wire up per-agent instruction files that all point at one source of truth (AGENTS.md).
  • Register MCP servers so agents can use the Angular CLI, Chrome DevTools and Figma.
  • Validate the setup by having an agent build a small feature and checking the output follows AGENTS.md and the style guide.

Prerequisites

  • Node.js 22+ and pnpm installed (npm i -g pnpm or Corepack).
  • The Angular CLI available (pnpm dlx @angular/cli@latest --version or a global install).
  • Git installed and configured (name + email).
  • At least one AI coding agent installed (e.g. Claude Code or Codex) – used in the final steps.
  • The project you chose in Lab 00, or permission to create a fallback workspace.
  • A terminal and your editor of choice (the lab assumes VS Code / JetBrains for the editor configs).
How to work through this lab

Each step has a Goal, some background, a Your task box, a collapsible Solution, and a Checkpoint to confirm you're done. Try the task yourself first; only then open the solution. From Step 7 onward, feel free to let your AI agent do the typing – that's the whole point.

Steps

  1. Scaffold the Angular project
  2. Clean up the initial commit & format with Prettier
  3. Install & set up Husky (pre-commit hook)
  4. Install Angular ESLint
  5. Add lint-staged
  6. Extend the ESLint configuration
  7. Add the Angular Coding Style Guide
  8. Write AGENTS.md and per-agent instruction files
  9. Register MCP servers
  10. Take it for a spin: validate the setup

5 Add lint-staged

Goal: lint and format only the staged files on commit – fast and focused.

Your task

  1. Install lint-staged.
  2. Configure which commands run for which file types in package.json.
  3. Point the Husky pre-commit hook at lint-staged.
Show solution 
shellpnpm add -D lint-staged

Add to package.json:

json"lint-staged": {
  "*.{html,js,ts}": ["eslint --fix", "prettier --write"],
  "*.{css,json,md,scss}": ["prettier --write"]
}

Replace the contents of .husky/pre-commit with:

shellpnpm lint-staged
Tip

Code files get eslint --fix then prettier; non-code files (CSS/JSON/MD/SCSS) only get Prettier. ESLint fixes logic/style; Prettier owns layout.

Checkpoint: staging a messy .ts file and committing auto-fixes and reformats it.

AGENTS.md (abbreviated – see the full version in this workspace):

markdownYou are an expert in TypeScript, Angular, and scalable web application
development. You write functional, maintainable, performant, and accessible
code following Angular and TypeScript best practices.

## Angular Best Practices
- Always use standalone components over NgModules
- Do NOT set `standalone: true` (it's the default in v20+)
- Use signals for state management
- Use `input()`/`output()` functions, not decorators
- Set `changeDetection: ChangeDetectionStrategy.OnPush`
- Use native control flow (`@if`, `@for`, `@switch`)
- Use `inject()` instead of constructor injection
- Use `NgOptimizedImage` for static images

## Accessibility Requirements
- MUST pass all AXE checks and WCAG AA minimums

## Project Style Guide
- Follow `style-guide/style-guide.md`.
- Load the specific guide for the file being touched:
  - TypeScript/Angular: `style-guide/style-guide.ts.md`
  - Templates: `style-guide/style-guide.html.md`
  - SCSS: `style-guide/style-guide.scss.md`
  - Dependencies: `style-guide/style-guide.npm.md`
  - Markdown: `style-guide/style-guide.md.md`
  - Git commits: `style-guide/style-guide.git.md`

10 Take it for a spin: validate the setup

Goal: prove the whole setup actually works – have an agent build a small feature, then confirm the generated code follows AGENTS.md and the style guide without you asking.

Everything so far was scaffolding. This step is the payoff. A good agentic setup shows itself when an agent, told only what to build, independently reaches for the conventions you encoded – signals, standalone, OnPush, inject(), native control flow – and the result sails through pnpm lint and the pre-commit hook with no hand-holding. If it doesn't, you've found exactly which rule to tighten.

Recap – the setup you're validating

The final commit ("finish ng agentic setup") pulled the agentic tooling together so every agent works from the same servers and rules: a hardened .aiignore (secrets, credentials and environment*.ts stay out of agent context); MCP servers defined once in .mcp.json and mirrored into .vscode/mcp.json, .junie/mcp/mcp.json and .codex/config.toml; thin per-agent files (.cursorrules, .junie/AGENTS.md) that defer to AGENTS.md, with .claude/settings.json enabling the project MCP servers; and an ng:update script for upgrades. Steps 1–9 built it – this step proves it works.

A counter that satisfies every check above (src/app/counter/counter.ts, inline template because the component is small):

tsimport { ChangeDetectionStrategy, Component, computed, signal } from '@angular/core';

@Component({
  selector: 'app-counter',
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    
Count: {{ count() }} ({{ isEven() ? 'even' : 'odd' }})


    
    
  `,
})
export class Counter {
  protected readonly count = signal(0);
  protected readonly isEven = computed(() => this.count() % 2 === 0);

  protected onIncrement(): void {
    this.count.update((value) => value + 1);
  }

  protected onDecrement(): void {
    this.count.update((value) => value - 1);
  }
}

Notice what the agent did without being told: OnPush, signals + computed(), protected readonly members, on-prefixed handlers, explicit : void return types, type="button", and the app- selector prefix. That's AGENTS.md and your ESLint config doing the work.

Why this is the real test

You're not done when the config files exist – you're done when an agent's output lands on-style on the first try. This experiment is how you know the setup is paying off, and it doubles as a regression test you can repeat whenever you change a rule.

When the agent gets it wrong

Treat a miss as a gap in the repo, not a one-off. Add or sharpen an ESLint rule, or add a line to the relevant style-guide file, then re-run the experiment. Encoding the rule once beats repeating it in every chat.

Checkpoint: an agent-generated feature passes pnpm lint, visibly follows AGENTS.md and the style guide, and was auto-formatted by the pre-commit hook on commit.

Definition of Done

You're finished when…

  • ✅ pnpm start serves the app and pnpm build succeeds.
  • ✅ pnpm format and pnpm lint both run clean (warnings OK, no errors).
  • ✅ Committing a messy staged file auto-fixes & reformats it via Husky + lint-staged.
  • ✅ style-guide/ has an index plus per-topic files, cross-linked.
  • ✅ AGENTS.md exists and every per-agent file defers to it.
  • ✅ .mcp.json registers the Angular CLI (and other) MCP servers; your agent can use them.
  • ✅ An agent-built feature lands on-style – follows AGENTS.md + the style guide and passes pnpm lint without manual fixups.

What each style-guide file enforces (highlights)

FileKey rules
ts.mdBoolean names start is/has/show; events start on; private + readonly by default; annotate public APIs, infer obvious locals; avoid lifecycle hooks, prefer afterNextRender(); Type[] over Array<Type>.
html.mdControl flow @if/@for; attribute order by category; [class.x]/[style.x] over ngClass/ngStyle; stable track; self-closing tags; max cyclomatic complexity 10.
scss.mdMax 2–3 nested selectors; CSS custom properties for tokens; :host for component knobs; @use/@forward; rem for typography; unitless zero.
a11y.mdWCAG 2.x AA, zero AXE violations; semantic HTML first; labelled controls; visible focus; contrast ≥ 4.5:1; never color alone.
spec.mdVitest unit (TestBed, provideZonelessChangeDetection(), test behavior not internals); Playwright e2e (role/label locators, web-first expect, axe checks).
git.mdConventional Commits (type(scope): description); defined types; atomic commits; LF endings; don't amend during review.
npm.mdCheck first-party first, license, pnpm audit; prefer maintained, low-dependency, TS-typed packages; commit lockfile; one package manager (pnpm).
md.mdOne H1; fenced + language-tagged code blocks; relative links; no bare URLs; descriptive image alt.

Command cheat sheet

shellng new ng-agentic --package-manager pnpm   # 1  scaffold
pnpm format                                  # 2  prettier --write .
pnpm add -D husky && pnpm exec husky init    # 3  pre-commit hook
ng add @angular-eslint/schematics            # 4  linting
pnpm add -D lint-staged                       # 5  staged-only fixes
ng lint --fix                                # 6  apply extended rules
# 7–9   create style-guide/, AGENTS.md + agent files, .mcp.json
pnpm lint && pnpm build                       # final verification
# 10    take it for a spin: agent-build a feature, review vs AGENTS.md
